Questions

  • What are the pro's & con's of local storage vs. server-side storage?
    Local Storage: It is fast, simple, but no scalability or availability. There is a limit to how many disks you can attached to a single system and how fast you can read from it via the controller If you system fails, it takes down the disks along with it. You can quickly bring up applications on a new system, but the data cannot be moved over quickly. Network Storage: It is sharable, scalable and highly available. You can access the data from multiple servers simultaneously.
  • When I should use local storage and database?
    You use Local Storage to handle information, that is only relevant to the particular client where you are running your web app or your mobile we app.
    On the other hand you use MySQL for permanent storage, that is, information that not only matters to your local user, but also to others.
    There must be a million ways to use each one, but to give you an example, I’d use local storage to save preference information about your user (Color Scheme/Skin of your App, Reminders, Personal Scores if you’re developing a game, etc.)
    You could then use MySQL to save information such as Prices for your Services/Products, Log Information, Financial transactions, etc. Source
